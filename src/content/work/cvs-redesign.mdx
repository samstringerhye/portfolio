# CVS Shop Website Redesign

Building foundations that gave teams freedom rather than constraints. CVS.com is one of the largest pharmacy e-commerce platforms in the US, and the platform had accumulated years of inconsistency across acquisitions. I owned the type system, the icon system, the component library, and the overall design direction: composable primitives rather than page-specific components, so teams could assemble freely.

  ## Typography & Icons

  I designed a type system with a rational scale and a grid-based icon and illustration system using CVS brand red. Each icon was constructed on a consistent grid so proportions held at any size. No one-off sizing decisions. The type scale and icon grid became the foundation everything else was built on.

<ProseGroup>

## The Problem

CVS sits at an unusual intersection: pharmacy and general retailer. The platform serves a wider range of intent than most e-commerce sites. Over time, the shop grew through acquisitions rather than deliberate system thinking, producing fractured type scales, inconsistent icons, and components built independently. Because CVS.com is live with real customer volume, I couldn't redesign from scratch. I had to build foundations that improved everything downstream without breaking what was already working.

</ProseGroup>

## The Work

I chose composable primitives over page-specific components. Rather than designing a "homepage hero" or a "category card," I built atomic pieces (type scale, icon grid, spacing system) that any team could assemble into what they needed. Starting with the typographic framework meant every downstream component inherited consistency. The icon system followed the same principle: every icon on a grid, using CVS brand red, proportions holding at any size without case-by-case adjustment.

</ProseGroup>

## What I Learned

Foundation work has the biggest downstream impact. When the typographic scale is resolved, product teams stop making ad hoc sizing decisions. When the icon system is consistent, no one debates stroke weight. Composable primitives unlocked speed: consistent inputs produce consistent outputs, freeing capacity for harder problems like checkout and pharmacy flows. The system gave teams freedom by removing the decisions that shouldn't vary.
